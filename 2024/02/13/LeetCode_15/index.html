LeetCode 15 三数之和

1 问题描述[1]

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

2 My Solution: 双重循环 + Dict + Set去重

这段代码实现了在给定整数数组中找到所有不重复的三元组，使得三元组的元素之和为零。以下是对代码的解释：

  1. 创建一个空列表 res 用于存储找到的三元组，以及一个空集合 unique_triplets 用于检测和避免重复的三元组。
  2. 构建字典 index_dict，其中键为数组元素，值为对应元素在数组中的最后一次出现的索引位置。
  3. 使用两个嵌套循环遍历数组 nums 中的元素，找到所有可能的两个元素的组合（nums[i] 和 nums[j]），并计算目标值 target，使得三者之和为零。
  4. 在 index_dict 中查找是否存在目标值 target，且其索引位置大于 j。
  5. 如果满足条件，则将三元组排序后转为元组，并检查是否已经在集合 unique_triplets 中，避免重复。如果不重复，则将三元组添加到结果列表 res 中，并将其加入集合 unique_triplets。
  6. 最终返回结果列表 res。

时间复杂度分析：

  • 构建字典 index_dict 的过程是 O(n)，其中 n 为数组的长度。
  • 两个嵌套循环的时间复杂度为 O(n^2)。
  • 在内层循环中，通过字典查询和集合操作的时间复杂度可以视为常数。
  • 因此，总体时间复杂度为 O(n) + O(n^2) = O(n^2)。

空间复杂度分析：

  • res 列表用于存储结果，其空间复杂度受到找到的三元组数量的影响，最坏情况下为 O(n)。
  • unique_triplets 集合用于存储已找到的三元组，其空间复杂度也受到找到的三元组数量的影响，最坏情况下为 O(n)。
  • index_dict 字典用于存储每个数字及其最后一次出现的索引位置，其空间复杂度为 O(n)。
  • 因此，总体空间复杂度为 O(n) + O(n) + O(n) = O(n)。

总体来说，该算法的时间复杂度是 O(n^2)，空间复杂度是 O(n)。

运行效率不能忍受。。。。

3 LeetCode Solution: 排序 + 双指针

这段代码实现了在排序后的数组中找到所有和为零的三元组的问题。下面是对代码的解释：

  1. res 是结果列表，用于存储所有满足条件的三元组。
  2. nums.sort() 对输入的整数数组进行排序。
  3. for i in range(len(nums) - 2): 开始遍历数组，注意这里遍历到倒数第三个元素，因为我们要找三元组。
  4. num_a = nums[i] 取得当前遍历到的元素。
  5. if num_a > 0: return res 如果当前元素 num_a 大于0，由于数组已排序，后续元素必然也大于0，因此可以直接返回结果列表。
  6. if i > 0 and num_a == nums[i - 1]: continue 跳过重复的元素，确保相同的 num_a 只被考虑一次。
  7. 初始化双指针 left 和 right，分别指向当前元素的下一个和数组的最后一个元素。
  8. prev_num_b = None 用于记录前一个 num_b 的值，避免重复。
  9. while left < right: 进入双指针遍历阶段，尝试找到满足条件的三元组。
  10. num_b, num_c = nums[left], nums[right] 取得当前左右指针指向的元素。
  11. total = num_a + num_b + num_c 计算三个元素的和。
  12. 如果 total == 0，则表示找到了一个满足条件的三元组：
    • if num_b != prev_num_b: 避免重复，只有当 num_b 不等于前一个 prev_num_b 时才将当前三元组加入结果列表。
    • res.append([num_a, num_b, num_c]) 将满足条件的三元组加入结果列表。
    • 更新 prev_num_b 为当前 num_b。
    • 分别将左指针 left 向右移动一位，右指针 right 向左移动一位。
  13. 如果 total > 0，说明当前三元组的和偏大，需要将右指针 right 向左移动一位。
  14. 如果 total < 0，说明当前三元组的和偏小，需要将左指针 left 向右移动一位。
  15. 最终返回结果列表 res，其中包含了所有满足条件的三元组。

时间复杂度分析：

  1. 排序阶段： nums.sort() 这一步的时间复杂度是O(nlogn)，其中n是输入数组的长度。
  2. 双指针遍历： 外部循环的时间复杂度是O(n)，内部的双指针遍历的时间复杂度是O(n)。由于内部循环是在外部循环内进行的，因此总体时间复杂度为O(n^2)。
  3. 总体时间复杂度： O(nlogn) + O(n^2) = O(n^2)（对于大的n，n^2的影响更显著）。

空间复杂度分析：

  1. 结果列表 res： 需要存储所有满足条件的三元组，最坏情况下有O(n)个三元组。因此，空间复杂度为O(n)。
  2. 除了结果列表外，只使用了常数级别的额外空间，因此对于每个输入大小n，空间复杂度为O(1)。

综上所述，代码的时间复杂度是O(n^2)，空间复杂度是O(n)。

运行效率瞬间舒服。。。。
